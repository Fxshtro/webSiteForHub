# План: Улучшения админ-панели

## Проблемы

1. **Сообщение "Введите корректные имя пользователя и пароль"** — появляется на логине и на странице участников проекта. Это стандартное сообщение Django при ошибке аутентификации Basic Auth. DRF `custom_exception_handler` из `hub/exceptions.py` оборачивает это сообщение и показывает даже в админке.

2. **Вкладка "Роли проектов" (Role)** — сейчас отдельная вкладка в админке. Роли проектов (дизайнер, разработчик, тестировщик и т.д.) должны добавляться только через инлайн внутри Проекта, не как отдельная вкладка.

3. **"Роли сайта" непонятно** — SiteRole сейчас называется просто "Роль сайта". Нужно добавить понятное описание: что это уровень доступа пользователя (Администратор, Лидер лаборатории, Менеджер проекта, Студент).

4. **"Файлы" непонятно** — File модель называется просто "Файлы". Нужно объяснить что это хранилище ссылок на внешние документы (Google Drive, Dropbox и т.д.), не загрузка файлов.

## Решения

### 1. Убрать ошибку аутентификации в админке

**Файл**: `hub/exceptions.py`

DRF exception handler оборачивает ВСЕ ошибки (включая Django admin auth errors) в кастомный формат. Нужно сделать проверку — если запрос идёт через DRF API (есть `context['view']`), оборачиваем. Если нет — пропускаем, оставляем стандартную Django обработку.

```python
def custom_exception_handler(exc, context):
    # Только для DRF API, не для Django admin
    if not context.get('view'):
        return None
    
    response = exception_handler(exc, context)
    if response is not None:
        response.data = {
            'error': True,
            'message': 'Ошибка запроса',
            'details': response.data
        }
    return response
```

### 2. Убрать вкладку "Роли проектов"

**Файл**: `hub/admin.py`

Добавить `Role` в `HIDDEN_MODELS` список. Роли проектов уже доступны через `ProjectRoleInline` внутри страницы Проекта.

```python
HIDDEN_MODELS = [..., Role]
```

### 3. Переименовать "Роли сайта" с пояснением

**Файл**: `hub/models.py`

```python
verbose_name = 'Уровень доступа'
verbose_name_plural = 'Уровни доступа (роли сайта)'
```

Добавить `help_text` в поле `title`:
```python
title = models.CharField(max_length=50, verbose_name='Название', help_text='Администратор — полный доступ, Лидер лаборатории — управление своей лаб., Менеджер проекта — управление проектом, Студент — только чтение')
```

### 4. Переименовать "Файлы" с пояснением

**Файл**: `hub/models.py`

```python
verbose_name = 'Ссылка на внешний файл'
verbose_name_plural = 'Ссылки на внешние файлы (Google Drive, Dropbox и т.д.)'
```

И в `hub/admin.py` тоже обновить заголовок вкладки FileAdmin.

## Файлы для изменения

| Файл | Изменение |
|---|---|
| `hub/exceptions.py` | Проверка context['view'] — пропускаем не-DRF ошибки |
| `hub/admin.py` | Добавить Role в HIDDEN_MODELS |
| `hub/models.py` | verbose_name для SiteRole, File; help_text для SiteRole.title |
